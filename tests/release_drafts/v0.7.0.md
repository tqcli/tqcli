# tqCLI 0.7.0 — TurboQuant Wheel Distribution

> Distributed on PyPI as **`turboquant-cli`** (import name unchanged: `import tqcli`).
> Apache-2.0. The umbrella package + `vllm-turboquant` are Apache-2.0; `llama-cpp-python-turboquant` inherits MIT from upstream.

## One-command install

**llama.cpp (any platform):**
```bash
pip install 'turboquant-cli[llama-tq]'
```

**vLLM (Linux + NVIDIA Ampere/Ada/Hopper):**
```bash
pip install 'turboquant-cli[vllm-tq]' \
  --find-links https://github.com/tqcli/vllm-turboquant/releases/latest
```

**Blackwell (sm_100 / sm_120 / sm_121 — DC, consumer, DGX Spark):**
```bash
pip install 'turboquant-cli[vllm-tq-blackwell]' \
  --find-links https://github.com/tqcli/vllm-turboquant/releases/latest
```

**macOS users:** install only `[llama-tq]` — there is no Darwin wheel for vLLM.

Verify: `python -c "import llama_cpp; print(llama_cpp.TURBOQUANT_BUILD)"` should print `True`.

## Highlights

- **TurboQuant fork wheels are now installable.** `llama-cpp-python-turboquant` ships
  on PyPI as a cibuildwheel matrix (Linux/macOS/Windows × Py 3.10–3.12 × CPU/CUDA/Metal).
  `vllm-turboquant` ships as a GitHub Release on `tqcli/vllm-turboquant`. Pinned to a
  golden commit AFTER Issue #22's four-patch page-size fix.
- **Engine Auditor** detects fork-vs-upstream at startup and prints a yellow Rich
  panel with the exact `pip install` command when capable hardware is running upstream.
  Stays silent on capable+fork installs and on hardware that cannot run TurboQuant.
- **CUDA 13.0 + Blackwell + PTX hedge for Rubin.** The vLLM build's
  `TORCH_CUDA_ARCH_LIST` is `"8.0 8.6 8.9 9.0 10.0 12.0 12.1+PTX"` —
  Ampere + Ada + Hopper + DC Blackwell + consumer Blackwell + DGX Spark / GB10 + PTX.
  CUDA 12.8 cannot compile sm_121, hence the toolkit bump.
- **GitHub Sponsors** wired via `.github/FUNDING.yml` -> `ithllc` (Ivey Technology Holdings LLC,
  the legal entity behind tqCLI).
- **Community verification** — `scripts/community_verify.sh` ships a consent-gated
  Mac-verifier flow with `--auto-report` (gh CLI) and `--manual` (paste-into-issue) modes.

## Breaking

- **PyPI distribution name changed from `tqcli` to `turboquant-cli`.** Import name is unchanged.
  The slug `tqcli` was unavailable on PyPI (taken by an unrelated project, TranQuant).
- **`pyproject.toml` extras renamed.** `[llama]` / `[vllm]` / `[all]` ->
  `[llama-tq]` / `[vllm-tq]` / `[all]`, plus the new `[vllm-tq-blackwell]`. The old
  keys no longer install anything; copy/paste from pre-0.7.0 docs will fail loudly.
- **No upstream `llama-cpp-python` / `vllm` install path.** tqCLI now ships exclusively
  against the TurboQuant forks. Users who want upstream behavior can swap in the
  upstream wheels manually, but the Engine Auditor will warn at every startup.
